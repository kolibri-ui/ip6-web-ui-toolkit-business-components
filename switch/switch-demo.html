Switch

A Switch uses the markup from a checkbox, but the interaction and design are styled like a toggle switch. A switch can have three states: on, off and indeterminate. It is possible for a developer to decide if the third state can be accessed.

A developer can decide about an id for a switch. An ID has to be unique. If no ID is supplied, it gets created automatically in the form switch-89bitdjyujhp. This id will change on each page refresh.

You can obtain the current state of the Switch Control bei checking the attached observable, which you have to supply while the instantiation of the Switch Control.

Examples

To create a Switch call the Switch Projector first define an options object. const options = {}; All not used Elements in the Options Object defaults to undefined. To change the Appearance and Initial State of the Switch, set the corresponding key in the Options Object. The following Options are available.

  • name (default: undefined, type: String)

    The Name for the Switch Control. If not supplied it will be generated. You better give the Switch a Name, because the generated name will be changed on each page load. This makes it harder to work with the value in the Backend.

  • id (default: undefined, type: String)

    The ID for the Input and Label of the Switch Control. If not supplied, it will be generated.

  • state (default: undefined, type: Boolean)

    The initial state for the Switch.

  • threeState (default: undefined, type: Boolean)

    If set to true, the Switch Control have a third state. This will then be indeterminate.

  • slim (default: undefined, type: Boolean)

    Set this option to true to get a Switch with Slim styling.

You can then call the Switch and supply the Observable from the Model as first Parameter and as the second Parameter the options Object. const demo = Switch(observable, options); It is also possible to insert the Options Object direct while calling the Switch() Projector. const demo = Switch(observable, { name: "demo-switch" });

Disabled, Required and Read Only

You can set Disabled, Required or ReadOnly directly on the Element with (disabled as example): const demo = Switch(observable, null, true);
demo.querySelector('input').disabled = true;
demo.classList.add('disabled');

Keyboard interaction

To change the state of a Switch you can use the keyboard. If a switch is focused and not readOnly, you can set the value with the arrow keys, with ← for off/false and → for on/true. If you are using a three state Switch, the state can be set to indeterminate by pressing the delete Key.

Initial States

The following table gives an overview about the possible state and default value combinations:

defaultState isThreeState Results in Remarks
false false Two State, current off
true false Two State, current on
null false Two State, current off per definition
false true Three State, current off

null true Three State, current indeterminate
