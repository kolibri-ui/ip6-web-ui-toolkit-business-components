Monolog

A Monolog displays Information to a user in a quick and easy fashion. It can be non-sticky or sticky. Besides this, it is also possible to display an extended Monolog with additional information about an error happend in the code. This is called a code-error Monolog. Try the different Monologs by pushing the buttons below.


How to use the Component

To use the Monolog Component instantiate first the MonologList component.

const monolog = MonologList();
As a return value you get the following Object: 
{
    list,
    info,
    success,
    warning,
    error
}
Append the received monolog.list() to the rootElement. 
rootElement.appendChild(monolog.list());
To supply a Monolog with all information needed, you have to create an options-object. The following Options are available:
  • title (default: "", type: String)

    The Title for the Monolog. This will be inserted into the first row of text of the Monolog.

  • message (default: undefined, type: String)

    Add some more Text to the Monolog. The length is not restricted, but less is more

  • sticky (default: false, type: Boolean)

    Should the Monolog be sticky? Set it to true.

  • attention (default: false, type: Boolean)

    Let the Monolog wiggle to get that extra bit of Attention.

  • codeError (default: undefined, type: String)

    If you add a String here, your Monolog will be of sort Code Error and the information supplied in this option will be copiable. If this option is set, the type of the monolog gets overridden with 'error'.

You can then emit the monolog ( here info as example) by calling one of the functions you got as a return value from the initialisation of the MonologList Component. 
monolog.info({
    title  : "Some Info",
    message: "And some Text"
});

Attention

You can use the Attention on each type of Monolog, but it makes at most sense on a type of error & sticky.

Just a Test
monolog.error({
    title    : "Some Error",
    message  : "Just a Test",
    sticky   : true,
    attention: true,
});
